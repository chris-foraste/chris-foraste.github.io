Throughout this project, we divided up the tasks based on our personal strengths. As I felt particularly skilled with the electronics content taught in the course, I took the lead on developing the local code (both for the driving and sensing subsystems) while also handling the wiring for the driving and power subsystems. It was actually easier than the tilting table maze which I had done earlier in the course. I also assisted in developing the communication code through pair programming and debugging. The other two members of the group, Leia and Vanessa, focused on internet-connected programming and the mechanical design of the sensing system respectively.

Code

This code block shows all of the code running on the Raspberry Pi, using the Flask framework. (The remaining code is simply an HTML webpage with a start button). I worked on the whole file and personally wrote the code up until the negotiation functions with the exception of several variable declarations used later on. An important feature I implemented was the control system which enabled the robot to adjust its speed in response to changes in pressure from the tube. Its design also makes it resistant to false alarms, as the sensitivity can be increased and the robot will ignore readings where both halves of the panel are triggered
