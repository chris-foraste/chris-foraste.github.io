For the final project of my Simple Robotics class, my group and I created a web-integrated robotic game. I focused on both the mechanical and software aspects of the project. I first constructed a cable management system to ensure that the cables leading from the hub to the parts wouldn't get twisted as the robot rotated. I also coded the code for a satisfying swinging motion on the local side. Finally, I integrated the local behavior onto the web by using a combination of Javascript and MicroPython.

Code Overview

A lot of the code was created for us in advance, however some further coding was needed. I wrote the local code for the robot swing, creating the timings and angles to create a realistic swinging motion. The code for that can be seen in the "Local Code" section above. Unfortunately due to the remote nature of the project that alone was not enough. I had to make the code runnable from a webpage sending a command. Webpages use JS as their scripting language while SPIKE 3 uses MicroPython. Thus to make the project work, I had to create a JavaScript function that takes in the power parameter and then outputs a properly formatted python code to execute. This was tricky not due to the complexity of the code but rather the difficulty in maintaining proper syntax for both of the languages. I will admit, I got caught up for quite a bit by missing semicolons and variable delcarations in my JS as I had not used that language in a while. However I was eventually able to complete it, finishing the project prototype.

During the protoype testing, we received the feedback that our product was unintuitive for users. To rectify this, we decided to make an instructions menu. With the help of the web developer bot, I created a single button that could open and close a menu. I also ensured that the text on the button in each state made sense to users. Finally, I used CSS to style the page, wrapping the controls in a clear background alongside enlarging the controls. I also used absolute positioning to place the button in the top right corner, making it's position clear and accessible for users.
